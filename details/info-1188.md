# INFO-1188: Bonjour - 32Bit entfernt, 64Bit erweitert

## Kategorie
**Change** | infoskop-Base-Updater | infoskop-Base-Setup

## Beschreibung

Die Installationsdatei für Bonjour 32bit wurde entfernt. Es gibt nun eine zweite Installationsdatei für Bonjour 64bit gleicher Version, aber im Praxistest funktionierte immer nur einer der Installer, wahrscheinlich abhängig von der Architektur (ARM und Intel Chips). Beide können gefahrlos in beliebiger Reihenfolge parallel installiert werden.

---

## Änderungen

### 1. Bonjour 32Bit entfernt

**Grund für Entfernung**:
- Moderne Systeme verwenden 64-Bit-Architekturen
- 32-Bit-Support wird nicht mehr benötigt
- Vereinfachung der Installationspakete
- Reduzierung der Paketgröße

**Ab Version 5.0.0**:
- Nur noch 64-Bit- und ARM-Systeme werden unterstützt
- 32-Bit-Systeme werden nicht mehr unterstützt

### 2. Zwei 64Bit Installer hinzugefügt

**Warum zwei Installer?**
- **Intel-Architektur**: x64-basierte Systeme
- **ARM-Architektur**: ARM64-basierte Systeme (z.B. neuere Surface-Geräte)

**Problem erkannt**:
- Im Praxistest funktionierte nicht immer derselbe Installer auf allen 64-Bit-Systemen
- Architektur-spezifische Unterschiede zwischen Intel und ARM

**Lösung**:
- Zwei separate 64-Bit-Installer bereitgestellt
- Beide können parallel installiert werden
- Automatische Auswahl des passenden Installers

---

## Installation

### Installationsverhalten

Beide Bonjour 64-Bit-Installer können:
- **Parallel installiert werden**: Keine Konflikte
- **In beliebiger Reihenfolge**: Reihenfolge spielt keine Rolle
- **Gefahrlos**: Kein Risiko von Überschreibungen

### Automatische Architektur-Erkennung

```
Installation:
1. System prüft Architektur (Intel/AMD oder ARM)
   ↓
2. Passender Bonjour-Installer wird ausgeführt
   ↓
3. Anderer Installer wird übersprungen (funktioniert nicht auf dieser Architektur)
   ↓
4. Bonjour erfolgreich installiert
```

## Technische Details

### Unterstützte Architekturen

| Architektur | Support | Bonjour Installer |
|-------------|---------|-------------------|
| 32-Bit (x86) | ❌ **Entfernt** | Nicht mehr verfügbar |
| 64-Bit Intel/AMD (x64) | ✅ | Bonjour64-Intel.msi |
| 64-Bit ARM (ARM64) | ✅ | Bonjour64-ARM.msi |

### Bonjour - Was ist das?

**Bonjour** (auch bekannt als mDNS/Zeroconf):
- Dienst für Netzwerkerkennung
- Automatisches Auffinden von Geräten im Netzwerk
- Wird von infoskop für Netzwerk-Kommunikation verwendet
- Ursprünglich von Apple entwickelt

**Verwendung in infoskop**:
- MediaServer-Erkennung im Netzwerk
- Multi-Client-Kommunikation
- Automatische Dienst-Discovery

## Kompatibilität

### Systemanforderungen (ab Version 5.0.0)

✅ **Unterstützt**:
- Windows 10/11 (64-Bit)
- Windows Server 2016/2019/2022 (64-Bit)
- ARM64-basierte Windows-Geräte

❌ **Nicht mehr unterstützt**:
- Windows 32-Bit (alle Versionen)
- Ältere 32-Bit-only-Systeme

### Migration von 32-Bit

Kunden mit 32-Bit-Systemen:
1. Müssen auf 64-Bit-System upgraden
2. Oder auf älterer infoskop-Version bleiben
3. Kontaktieren Sie Support für Migrationsberatung

## Installation - Schritt für Schritt

### Automatische Installation (via Updater/Setup)

```
1. infoskop-Base-Updater startet
   ↓
2. Prüft installierte Bonjour-Version
   ↓
3. Wenn nicht vorhanden oder veraltet:
   - Startet Bonjour64-Intel.msi
   - Startet Bonjour64-ARM.msi
   ↓
4. System wählt automatisch passenden Installer
   ↓
5. Bonjour installiert und konfiguriert
```

### Manuelle Installation

Falls manuelle Installation erforderlich:

**Für Intel/AMD-Systeme**:
```cmd
msiexec /i Bonjour64-Intel.msi /quiet
```

**Für ARM-Systeme**:
```cmd
msiexec /i Bonjour64-ARM.msi /quiet
```

**Beide parallel** (empfohlen - automatische Auswahl):
```cmd
msiexec /i Bonjour64-Intel.msi /quiet
msiexec /i Bonjour64-ARM.msi /quiet
```

## Fehlerbehebung

### Problem: Bonjour funktioniert nicht

**Diagnose**:
1. Öffnen Sie Windows Services (`services.msc`)
2. Suchen Sie nach "Bonjour-Dienst" oder "Bonjour Service"
3. Prüfen Sie Status (sollte "Running" sein)

**Lösung**:
```
Wenn Dienst nicht läuft:
1. Rechtsklick auf "Bonjour Service"
2. "Starten" wählen
3. Starttyp auf "Automatisch" setzen
```

### Problem: Falscher Installer wurde verwendet

**Symptom**: Installation schlägt fehl oder Bonjour funktioniert nicht

**Lösung**:
- Installieren Sie beide 64-Bit-Installer
- System wählt automatisch den passenden
- Neustart des Systems nach Installation

## Architektur-Erkennung

Das System erkennt automatisch:
```
Windows-Architektur prüfen:
- PowerShell: (Get-WmiObject Win32_OperatingSystem).OSArchitecture
- CMD: wmic os get osarchitecture
- Programmatisch: Environment.Is64BitOperatingSystem

Ergebnis:
"64-bit" → Intel/AMD x64 → Bonjour64-Intel
"ARM64" → ARM-Architektur → Bonjour64-ARM
```
