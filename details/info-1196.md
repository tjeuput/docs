# INFO-1196: Infoskop-Client - Webframe Ladeanzeige

## Kategorie
**Enhancement** | Infoskop-Client.exe

## Beschreibung

Für die verzögerte Anzeige der Webframes (infoskop-Monitor und infoskop::LEA) wurde eine „Bitte warten …" Anzeige bereitgestellt.

### Problem

Bei längeren Ladezeiten der Webframes:
- **Keine Rückmeldung**: Benutzer sahen einen leeren Bildschirm
- **Verwirrung**: Unklar, ob die Anwendung noch lädt oder abgestürzt ist
- **Schlechte UX**: Keine Information über den Ladestatus
- **Ungeduld**: Benutzer klickten mehrfach oder brachen ab

Dies betraf insbesondere:
- infoskop-Monitor beim ersten Start
- infoskop::LEA beim Öffnen
- Langsame Netzwerkverbindungen
- Systeme mit begrenzten Ressourcen

### Lösung

Eine transparente Ladeanzeige wurde implementiert:
- **"Bitte warten …" Anzeige**: Wird während des Ladevorgangs angezeigt
- **Feedback**: Benutzer wissen, dass die Anwendung arbeitet
- **Professionell**: Saubere, informative Anzeige
- **Verschwindet automatisch**: Nach erfolgreichem Laden der Webframes

## Technische Details

### Implementierung
- **Anzeige-Komponente**: Dedizierter Lade-Dialog
- **Trigger**: Aktiviert bei Webframe-Initialisierung
- **Auto-Hide**: Verschwindet nach erfolgreichem Laden
- **Error-Handling**: Bei Ladefehlern erscheint Fehlermeldung statt Ladeanzeige

### Ladeszenarien

| Szenario | Ladezeit | Verhalten |
|----------|----------|-----------|
| Lokale Installation | < 2s | Kurzes "Bitte warten …" |
| Netzwerk-Installation | 2-5s | Anzeige während gesamter Ladezeit |
| Langsame Verbindung | > 5s | Anzeige + ggf. Fortschrittsbalken |
| Ladefehler | - | Fehlermeldung statt Ladeanzeige |

Das neue Produkt profitiert besonders von dieser Verbesserung:
- **Erstmaliges Laden**: Kann etwas länger dauern
- **Lizenzprüfung**: Läuft im Hintergrund während Laden
- **Automatische Aktivierung**: Bei korrekt hinterlegtem Leistungskatalog

## UX-Verbesserung im Detail

### Vorher (ohne Ladeanzeige)
```
1. Benutzer klickt auf Monitor/LEA
2. Leerer Bildschirm (3-5 Sekunden)
3. ??? (Benutzer unsicher)
4. Webframe erscheint plötzlich
```

### Nachher (mit Ladeanzeige)
```
1. Benutzer klickt auf Monitor/LEA
2. "Bitte warten …" Anzeige erscheint sofort
3. Benutzer weiß: Anwendung lädt
4. Anzeige verschwindet automatisch
5. Webframe ist bereit
```

---

## Konfiguration

Die Ladeanzeige ist standardmäßig aktiviert und erfordert keine Konfiguration. Sie erscheint automatisch bei allen Webframe-Ladevorgängen.

---

## Performance-Hinweise

Die Ladezeit der Webframes kann beeinflusst werden durch:
- **Netzwerkgeschwindigkeit**: Besonders bei Netzwerk-Installationen
- **System-Ressourcen**: CPU und RAM-Verfügbarkeit
- **Erste vs. wiederholte Starts**: Caching kann spätere Starts beschleunigen
- **Anzahl gleichzeitiger Clients**: Server-Last bei mehreren Benutzern
